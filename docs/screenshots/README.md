# Screenshots

The main `README.md` references the images below. They are **not committed yet** —
capture them from a running instance (`npm run dev`, seeded library) and save them here
with these exact filenames so the README renders them.

| Filename                | What to capture                                                                                                                                                                                              |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `reading-workspace.png` | Reading mode (`/papers/[slug]/read`) with all three panes visible: structured notes (left), the PDF (centre), the assistant rail (right). Ideally show a passage summary and one annotation/Q&A in the rail. |
| `selection-ask.png`     | A text selection in the PDF with the floating "Ask about this / Note" toolbar visible (or the small ask composer open).                                                                                      |
| `dashboard.png`         | The dashboard (`/dashboard`) showing the stat strip, the 14-day activity bars, and the primary work cards.                                                                                                   |
| `radar.png`             | Research Radar (`/radar`) after a refresh, showing a few candidate cards with their "why it matters" explanations.                                                                                           |

Guidance:

- Use a wide viewport (≥ 1440px) so the three-pane workspace isn't collapsed to tabs.
- Prefer light mode for legibility on the README, or capture both and suffix `-dark`.
- Crop to the app content (no OS chrome). PNG, roughly 1600px wide is plenty.
- Do not include any confidential material — use the seeded public papers only.
